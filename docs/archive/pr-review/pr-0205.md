# PR #205: fix: ExpandableColumnCard ignores DataStore persisted state

**Goal:** Card collapse/expand preferences weren't persisting. `LaunchedEffect(expanded)` fired on first composition with default value, overwriting saved DataStore preferences. `remember { mutableStateOf(startExpanded) }` never updated when DataStore loaded real value. Fix: `LaunchedEffect(startExpanded)` syncs external state changes (DataStore → local state). Click handlers explicitly call `onExpandedChange(expanded)`.

**Files:** `AndroidGarage/androidApp/.../ExpandableColumnCard.kt`, `Main.kt`.

**Rationale:** Classic Compose gotcha — `LaunchedEffect(localState)` writes default-before-DataStore; `LaunchedEffect(externalKey)` is the correct sync direction.

**Still-current lesson:** In Compose, `LaunchedEffect` *key* controls direction of sync. Key the effect on the external source so external→local sync fires when the external value arrives; keying on local state writes default back and clobbers it.
