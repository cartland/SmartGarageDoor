# PR #285: fix: FCM registration retry when NOT_REGISTERED

**Goal:** `FcmRegistration` composable now actually calls `registerFcm()` when status is NOT_REGISTERED (previously only logged "registering..." with no action). Extract decision logic into `FcmRegistrationStatus.toAction()` for testability. 3 tests verify correct action per status.

**Notes:** Both `FcmRegistration` composable and `FcmRegistrationAction` were later deleted in PR #289 when FCM registration moved to app-scoped `FcmRegistrationManager`.

**Files:** `AndroidGarage/androidApp/fcm/FcmRegistration.kt`, `usecase/.../FcmRegistrationAction.kt` (new, later deleted), `FcmRegistrationActionTest.kt` (new).
