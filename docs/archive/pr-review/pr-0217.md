# PR #217: refactor: Replace AppLogger wrapper with standalone export function

**Goal:** Delete `AndroidAppLoggerRepository` interface + impl + DI binding. Add `AppLogEvent.toCsvRow()` in shared domain model (KMP-compatible). Add `exportAppLogCsvToUri()` standalone function (15 lines of Android I/O). The wrapper existed solely for `writeCsvToUri`; CSV export is presentation-layer action, not data access.

**Files:** delete `AppLoggerRepository.kt`, new `ExportAppLogCsv.kt`, `domain/.../AppLogEvent.kt`, `AppComponent.kt`, `LogSummaryCard.kt`, `FakeAppLoggerRepository.kt`.

**Rationale:** The "interface exists solely for one Android-specific method" smell was real; CSV export is a presentation action, not data access. Removing the wrapper and keeping the CSV row as shared is correct.

**Still-current lesson:** A repository interface that wraps exactly one Android-specific API call is a DI shim, not a repository. Inline the action as a standalone function and save the abstraction budget for something that has multiple impls.
