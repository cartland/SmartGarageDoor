# PR #22: fix: Disable HTTP body logging in release builds

**Goal:** Gate `HttpLoggingInterceptor` level on `BuildConfig.DEBUG`. Debug: `Level.BODY` for development. Release: `Level.NONE` to prevent auth tokens from leaking to Logcat. Previously `Level.BODY` was unconditional. Phase 5.2.

**Files:** `AndroidGarage/androidApp/.../internet/GarageNetworkService.kt`.

**Rationale:** Security fix — HTTP body logging was leaking auth tokens to Logcat in release builds. Gate on `BuildConfig.DEBUG` is still the rule.

**Still-current lesson:** Any log that prints HTTP bodies, headers, or auth tokens must be gated on `BuildConfig.DEBUG`. Default-on logging is a secret-leak waiting for a production build.
