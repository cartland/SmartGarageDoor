**Rationale:** "Null-initial, render once real value arrives" is the right shape for DataStore-backed state. Avoids the "correction animation" from the default value.

**Still-current lesson:** For async-loaded UI state (DataStore/Room/network), pick a nullable type and skip rendering until the real value arrives — a default-then-correct animation reveals "we don't know yet" and looks like a bug.
