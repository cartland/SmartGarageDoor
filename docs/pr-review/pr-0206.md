# PR #206: fix: Block git push to branches with auto-merge enabled

**Goal:** Add PreToolUse hook check: before `git push`, query if target branch's PR has auto-merge enabled. If active, push blocked with instructions to disable auto-merge first. Prevents race condition where squash merge executes before new commits arrive, silently losing them.

**Files:** `.claude/hooks/git-guardrails.sh`.

**Rationale:** The auto-merge race is invisible — commits silently lost when squash executes before push arrives. A PreToolUse hook that blocks the push is the right layer. Hook still active, rule still in memory.

**Still-current lesson:** If `--auto` merge + a push on the same branch can drop commits, block the push at the tool-use layer — don't rely on the author remembering to disable auto-merge. Hooks are where anti-footgun rules live.
