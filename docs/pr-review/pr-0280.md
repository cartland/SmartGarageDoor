# PR #280: fix: Add yield() after PushStatus.SENDING in production repository

**Goal:** Same StateFlow conflation fix as PR #279 but for production `NetworkRemoteButtonRepository`. Without `yield()`, if `getServerConfigCached()` returns from cache and HTTP is fast, `SENDING` can be conflated before the state machine sees it — no network timeout starts and UI skips `SendingToDoor`. One-line fix: `yield()` after `_pushButtonStatus.value = PushStatus.SENDING`.

**Notes:** Both the yield workaround and the entire PushStatus StateFlow were removed in PR #282.

**Files:** `AndroidGarage/data/src/commonMain/kotlin/com/chriscartland/garage/data/repository/NetworkRemoteButtonRepository.kt`.
