# PR #295: fix: Auth state UI regression — replace stateIn(Eagerly) with explicit collect

**Goal:** UI elements depending on sign-in state were not updating on real devices after sign-in/sign-out. Root cause: `stateIn(viewModelScope, Eagerly, Unknown)` in `DefaultAuthViewModel` had subtle timing issues where the internal collector didn't reliably propagate upstream `AuthState` changes to Compose's `collectAsState`. Fix: switch to explicit `MutableStateFlow + viewModelScope.launch.collect` pattern (matches `DoorViewModel.currentDoorEvent` which works reliably). New `authState_reflectsRepositoryChanges` test verifies propagation.

**Notes:** The symptoms here were later root-caused in PR #328 as a separate issue (imperative `getIdToken(forceRefresh=true)` network race in the repository layer). This PR's change became the ADR-017 Rule 6 template.

**Files:** `AndroidGarage/usecase/.../AuthViewModel.kt`, `DefaultAuthViewModelTest.kt`.
