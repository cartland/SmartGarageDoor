# PR #281: docs: ADR-013 — Flow and StateFlow boundaries

**Goal:** Document architectural decision restricting StateFlow to the ViewModel layer. UseCases: `suspend fun`: `AppResult<D, E>` or `fun`: `Flow<T>`. Repositories: `Flow<T>` for streams, `suspend fun` for actions. StateFlow below ViewModel requires individual approval. Lists 3 current violations pending review. Supersedes StateFlow guidance in ADR-010.

**Notes:** Direction later reversed by ADR-022 (#357) — StateFlow for state-y data lives at the repository boundary, not the ViewModel.

**Files:** `AndroidGarage/docs/DECISIONS.md`.
