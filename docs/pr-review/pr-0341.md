# PR #341: fix: Fetch snooze status on ViewModel init

**Goal:** SnoozeState was stuck on `Loading` because the ViewModel never triggered the initial fetch — only a 60s polling `LaunchedEffect` in `ProfileContent` eventually populated it. Card showed "Door notifications" (Loading text) instead of "Door notifications enabled." Add init fetch in the ViewModel, same class as ADR-018 auth state issue: populate eagerly, don't rely on UI-layer polling. `initFetchesSnoozeStatus` test added.

**Notes:** The ViewModel-init fetch approach here was later replaced in #344 by moving the init fetch to the repository's `externalScope` to avoid VM cancellation stranding the state.

**Files:** `AndroidGarage/usecase/.../RemoteButtonViewModel.kt`, `RemoteButtonViewModelTest.kt`.
