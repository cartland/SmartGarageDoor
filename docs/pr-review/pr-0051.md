# PR #51: docs: Add ADR-008 for parsing objects over generic extensions

**Goal:** Document preference for parser objects over extension functions on generic receiver types. Example: `FcmPayloadParser.parseDoorEvent(data)` instead of `Map<K, V>.asDoorEvent()`. Migration note: only refactor when safe, especially FCM code.

**Notes:** ADR-008 was later renumbered to ADR-009 when the naming convention ADR took #008 in PR #133.
