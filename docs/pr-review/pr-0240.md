# PR #240: refactor: AppLoggerViewModel uses LogAppEventUseCase + ObserveAppLogCountUseCase

**Goal:** First step of Phase 43 (ViewModel→UseCase enforcement). `AppLoggerViewModel` no longer depends on `AppLoggerRepository` directly. New UseCases: `LogAppEventUseCase` wraps `appLoggerRepository.log(key)`, `ObserveAppLogCountUseCase` wraps `appLoggerRepository.countKey(key)`. Both thin wrappers — intentional. Per architecture, all repository access from ViewModels must go through a UseCase even when trivial.

**Files:** `AndroidGarage/usecase/.../AppLoggerViewModel.kt`, `LogAppEventUseCase.kt` (new), `ObserveAppLogCountUseCase.kt` (new), `DefaultAppLoggerViewModelTest.kt`, `AppComponent.kt`.
