# PR #233: fix: Snooze Loading state stuck forever on fetch failure

**Goal:** If first snooze fetch failed, `SnoozeState.Loading` persisted forever. Fix: `clearLoadingState()` transitions to `NotSnoozing` on failure, but only if still `Loading` (subsequent poll failures keep last known state). Two new tests.

**Files:** `AndroidGarage/data/src/commonMain/kotlin/com/chriscartland/garage/data/repository/NetworkSnoozeRepository.kt`, `PushRepositoryTest.kt`.
